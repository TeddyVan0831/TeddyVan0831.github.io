**4 月 8 日，腾讯正式发布AI 浏览器 QBotClaw。**

这是继美团 Tabbit 之后，又一款打上"AI 浏览器"标签的产品。

先说结论：**功能有价值，但"龙虾"这个名字，有蹭 OpenClaw 热度之嫌。**

---

## 一、QBotClaw 是什么？

根据腾讯官方发布的信息，QBotClaw 的核心功能有 5 个：

**1. 多 API 自由配置**

用户可接入国内主流大模型 API Key，不再被单一模型绑定。

**2. QQ 浏览器 Skill 集成**

内置 QQ 浏览器原生 AI 能力。

**3. OpenClaw 生态兼容**

全面兼容 OpenClaw 技能体系。

**4. 跨端远程控制**

微信扫码绑定 Clawbot，实现移动端操控电脑端。

**5. X5use 视觉识别**

高精度视觉感知，支持复杂页面理解。

目前 Mac 版已上线，Windows 版即将推出。

---

## 二、腾讯的"五只钳子"

很多人可能不知道，腾讯基于 OpenClaw 或兼容 OpenClaw 生态的产品，**不是一个，而是五个**。

让我们来看一下完整的产品矩阵：

| 产品                 | 定位                            | 入口              | 发布时间           |
| ------------------ | ----------------------------- | --------------- | -------------- |
| **QClaw**          | 个人远程控制电脑，免费，微信直连，零门槛          | 个人微信            | 2026 年 3 月（内测） |
| **WorkBuddy**      | 企业办公 AI 助手，安全合规，预置 22,000+ 技能 | 企业微信/个人微信/飞书/钉钉 | 2026 年 3 月 9 日 |
| **QQ 接入 OpenClaw** | QQ 社交入口，海量用户，无需注册             | QQ              | 2026 年 3 月 7 日 |
| **QBotClaw**       | 浏览器自动化，Mac 版已上线               | QQ 浏览器          | 2026 年 4 月 8 日 |
| **ClawPro**        | 企业级 AI 智能体管控平台，10 分钟部署        | 云端部署            | 2026 年 4 月 2 日 |

⚠️ **重要澄清**：QQ 接入 OpenClaw 不是腾讯自研的独立产品，而是腾讯 QQ 平台在**2026 年 3 月 7 日**开放原生集成 OpenClaw，让用户可以在 QQ 里创建机器人接入 OpenClaw 生态。

这意味着什么？

腾讯在短短一个月时间内（3 月 7 日 -4 月 8 日），密集发布了 5 款与 OpenClaw 相关的产品。

这个节奏，值得注意。

## 三、VS 美团 Tabbit：同台打擂

说到 AI 浏览器，不得不提一个月前美团推出的**Tabbit**。

根据 36 氪报道，3 月 2 日，美团光年之外团队发布 Tabbit AI 浏览器，定位为"OpenClaw 的最佳拍档"。

两款产品有何异同？让我们直接上对比：

| 维度       | QBotClaw          | Tabbit          |
| -------- | ----------------- | --------------- |
| **发布时间** | 2026 年 4 月 8 日    | 2026 年 3 月 2 日  |
| **出品方**  | 腾讯                | 美团光年之外          |
| **产品定位** | 开放 API+ 浏览器自动化    | 轻量级 AI 原生浏览器    |
| **核心卖点** | 多模型自由切换           | 零门槛免费使用         |
| **技能生态** | 兼容 OpenClaw 生态    | 定位为 OpenClaw 补充 |
| **用户成本** | 自备 API Key        | 完全免费            |
| **平台覆盖** | Mac 已上，Windows 待发 | 多平台             |

**Tabbit 的核心优势**：

- 完全免费，内置模型无需付费
- 零门槛，公测无需邀请码
- 专注轻量级任务（标签整理、内容分析、跨平台发布）
- 被定位为 OpenClaw 的"轻量搭档"

**QBotClaw 的核心优势**：

- 高度开放，不绑定单一模型
- 可复用已有 API Key，成本可控
- 兼容 OpenClaw 生态，技能丰富
- 跨端协同能力强（微信控制电脑）

**简单来说**：

如果你已有国内大模型 API Key，QBotClaw 的灵活性更有价值。

如果你不想折腾，Tabbit 的零门槛更友好。

## 四、重点来了：它不是龙虾，只是穿着龙虾外衣

**这里我必须说一个问题：QBotClaw 这个名字，有蹭热度之嫌。**

### "龙虾"是谁的？

"龙虾"这个概念，最早来自**OpenClaw**，2025 年 11 月由奥地利程序员彼得·斯坦伯格创建的开源 AI Agent 项目，核心理念是"让 AI 从聊天进化到干活"。

但腾讯 QBotClaw 呢？

它叫"浏览器龙虾"，却与 OpenClaw 没有直接关系。它是腾讯自研的产品，却顶着"龙虾"的帽子。

### 套娃营销之嫌

腾讯的"龙虾"矩阵，包括 QClaw、WorkBuddy、QQ 接入 OpenClaw、浏览器龙虾——每一只都叫龙虾。

但问题是：

- OpenClaw 是一只真正的"钳子"（Claw），代表 AI 执行操作的能力
- 腾讯的"龙虾"只是借用了这个符号

**这不是真正的龙虾，这是一只穿着龙虾外衣的螃蟹。**

### 功能是好的，名字是蹭的

**客观地说，QBotClaw 的产品功能是有价值的。**

多 API 自由配置，解决了用户被单一模型绑定的问题。

OpenClaw 生态兼容，快速获取丰富技能。

跨端协同能力（微信控制电脑）确实实用。

这些都是实打实的功能，不是噱头。

**但"龙虾"这个名字，确实有点尴尬。**

如果腾讯一开始就叫它"QQ 浏览器 AI Plus"或"QBot 浏览器"，可能更合适。

现在顶着"龙虾"的名头，容易让用户混淆：

- 这是 OpenClaw 的腾讯版吗？
- QClaw 和 QBotClaw 是什么关系？
- QBotClaw 需要单独的硬件进行私有化部署吗？还是跑在腾讯的云主机上？
- QBotClaw 能 7*24 小时干活吗？

**蹭热度一时爽，认知混乱火葬场。**

---

## 五、AI 浏览器的真正价值

撇开命名问题，AI 浏览器赛道本身值得关注。

浏览器是互联网最基础的入口，近 30 年来形态变化有限。

**AI 浏览器的意义**：让浏览器从"信息窗口"升级为"任务执行器"。

传统浏览器：

> 用户→打开网页→手动操作→获取信息

AI 浏览器：

> 用户→下达指令→AI 自动操作→完成任务

**这是真正的范式转换。**

无论是 QBotClaw 还是 Tabbit，都在推动这个趋势：

- Tabbit 解决的是"轻量级信息处理"问题
- QBotClaw 解决的是"多模型灵活调度"问题

两者不是非此即彼，而是面向不同用户需求。

## 六、给用户的建议

**腾讯 Claw/Buddy 家族选型指南**：

| 如果你是...         | 推荐产品          | 原因           |
| --------------- | ------------- | ------------ |
| 普通用户，想零门槛体验     | **QClaw**     | 完全免费，微信直连    |
| 企业用户，注重安全合规     | **WorkBuddy** | 企业级安全，集成企微   |
| 有 API Key，追求灵活性 | **QBotClaw**  | 多模型自由切换      |
| 企业 IT 管理员       | **ClawPro**   | 10 分钟部署，统一管控 |

**如果想体验 AI 浏览器**：

1. **首选 Tabbit**：完全免费，零门槛，适合轻量级需求
2. **如果你有 API Key**：QBotClaw 的灵活性更有价值
3. **如果你是腾讯全家桶用户**：QBotClaw 与微信/QQ 的协同更顺畅

**但请注意：AI 浏览器是效率工具，不是万能助手。**。

复杂任务仍需人工判断。

AI 执行结果需要审核。

隐私安全需要警惕。

**但"龙虾"这个名字，我给它打 7 分——蹭热度扣 3 分。**

希望腾讯在后续迭代中，能建立起自己清晰的品牌认知。

毕竟，**真正的龙虾只有一只**。
